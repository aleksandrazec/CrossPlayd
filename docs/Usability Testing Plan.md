#### Assessment Testing Plan

##### Objectives and Methods

Our objectives are defined as:

* Understand if something confuses users while using our prototype
* Test alternative solutions
* Understand user needs and requirements

The testing method that was used: moderated, in-person and qualitative.

##### Subjects and Screening Questions

We will choose subjects based on these two screening questions:

* Do you play video games regularly?
* Do you read online video game forums and reviews?

##### Defining Tasks

Tasks for Assessment Testing:

* Task 1 - Log into the application.
* Task 2 - Navigate to Games Tab.
* Task 3 - Add Resident Evil Requiem to your Library with Status Playing and Rating of 8 stars.
* Task 4 - Add a review to Resident Evil Requiem.
* Task 5 - View Resident Evil Requiem in your Library.
* Task 6 - Navigate to the Community Tab.

The starting point of Task 1 is the Home page of our application. After the first task, all subsequent tasks start subsequently, and each starting point is the last screen in the previous test.

##### Post-Test Debrief

Additional Comments from users:

* User 1 - Thought that the ratings should be switched and presented from lowest to highest in Task 3.
* User 2 - Said that the search functionality, which was not fully implemented in the prototype, would make it easier to search for games.
* User 3 - Thought that the design was nice and the features were well implemented.
* User 4 - Said the layout of the account page was a little confusing and it would be better if it was more intuitive.
